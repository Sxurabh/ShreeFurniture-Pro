# Coding Standards

## General
- TypeScript strict mode
- Server actions for mutations
- Zod validation required

## UI & Design System
- TailwindCSS for styling
- Shadcn/ui for core components
- Lucide React for icons
- Reusable, accessible components (WCAG compliant)
- Mobile-first responsive design

## File Conventions
- Use `kebab-case` for file/folder names
- Place UI components in the `components/` directory
- Group related features together when applicable
